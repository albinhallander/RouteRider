3. **Confirm with the user** before pushing. Skip this only if the user said "push without asking" or used a one-shot phrase like "just push it".

4. **Push.** `git push origin main`. Never use `--force` or `--no-verify` unless the user explicitly asks.

5. **Report CI status.** Run `gh run list --branch main --limit 1` (or `gh run watch` if the user wants to wait). Relay the result. If CI fails, offer to pull the failure log with `gh run view <id> --log-failed`.
